Introduction

Ticket Master API Documentation (V1)

Base URL: http://localhost:8000

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting logging in to the application.

Authentication

User Authentication

Login User

Authenticate a user and return a new bearer token.

Example request:
curl --request POST \
    "http://localhost:8000/api/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"fernando.anderson@example.net\",
    \"password\": \"G_0^Eo\\/1Y\\/$}]b$~`Z\"
}"

const url = new URL(
    "http://localhost:8000/api/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "fernando.anderson@example.net",
    "password": "G_0^Eo\/1Y\/$}]b$~`Z"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (201):


{
    "message": "manager@example.com",
    "status": 200,
    "data": {
        "token": "YOUR_AUTH_KEY"
    }
}

Logout User

requires authentication

Sign out the authenticated user and delete their assigned bearer token.

Example request:
curl --request POST \
    "http://localhost:8000/api/logout" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost:8000/api/logout"
);

const headers = {
    "Authorization": "Bearer {YOUR_AUTH_KEY}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "POST",
    headers,
}).then(response => response.json());

Author(s)

Get Author(s)

requires authentication

Return all authors registered in the system. Results can be filtered and sorted.

Example request:
curl --request GET \
    --get "http://localhost:8000/api/V1/authors?filter%5Bid%5D=filter%5Bid%5D%3D1%2C2%2C3&filter%5Bname%5D=filter%5Bname%5D%3D%2Amanager%2A&filter%5BcreatedAt%5D=filter%5BcreatedAt%5D%3D2020-01-01%2C2020-01-02&filter%5BupdatedAt%5D=filter%5BupdatedAt%5D%3D2020-01-01%2C2020-01-02&sort=sort%3D-name%2Cemail" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost:8000/api/V1/authors"
);

const params = {
    "filter[id]": "filter[id]=1,2,3",
    "filter[name]": "filter[name]=*manager*",
    "filter[createdAt]": "filter[createdAt]=2020-01-01,2020-01-02",
    "filter[updatedAt]": "filter[updatedAt]=2020-01-01,2020-01-02",
    "sort": "sort=-name,email",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Authorization": "Bearer {YOUR_AUTH_KEY}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());
